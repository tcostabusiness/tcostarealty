# TCostaRealty.com — Boutique GitHub Pages Website

This folder is ready to upload to a new GitHub repository named `tcostarealty`.

## Included

- `index.html` — complete one-page website
- `styles.css` — responsive boutique styling
- `script.js` — mobile menu and contact-form behavior
- `assets/tyler-costa-headshot.png` — Tyler's headshot with the background removed
- `assets/island-line.svg` — decorative original artwork
- `.nojekyll` — tells GitHub Pages to serve the files directly

## Before publishing

### 1. Add Tyler's business email

Open `script.js` and replace:

`YOUR_EMAIL@example.com`

with the real business email address.

### 2. Add the approved eXp logo

The site currently uses a clean text disclosure: **Brokered by eXp Realty**.

For logo use, download the current approved eXp Realty logo from your official eXp Canva Brand Kit. Save it as:

`assets/exp-realty-logo.png`

Do not use a logo copied from Google Images or a recreated logo. After you provide the approved file, the HTML can be updated to display it.

### 3. Review brokerage compliance before advertising

Submit the final website to your broker or eXp marketing/compliance channel if required. Confirm your exact advertised name, brokerage disclosure, contact details, fair-housing disclosures, and any office information required by your brokerage.

## Fresh GitHub setup

1. Create a new public repository named `tcostarealty`.
2. Choose **Add file → Upload files**.
3. Upload the **contents inside this folder**, not the folder itself.
4. Confirm `index.html` appears in the repository's top level.
5. Go to **Settings → Pages**.
6. Under Source, choose **Deploy from a branch**.
7. Choose `main` and `/(root)`, then Save.
8. Wait for the Pages deployment to finish.
9. Test: `https://tcostabusiness.github.io/tcostarealty/`

## Important: custom domain

This package intentionally contains **no `CNAME` file**. Test the GitHub Pages address first. Connect `TCostaRealty.com` only after the GitHub address works correctly.
